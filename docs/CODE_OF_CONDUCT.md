# Code of Conduct

## Our Commitment

The Aura AI team is committed to creating a respectful, collaborative, and productive project environment.

---

## Expected Behavior

Team members and contributors should:

- communicate respectfully
- ask before making major structural changes
- give constructive feedback
- document important decisions
- keep discussions project-focused
- help other team members understand changes
- avoid unnecessary blame
- respect project ownership and responsibilities

---

## Unacceptable Behavior

The following are unacceptable:

- harassment or personal attacks
- insulting team members
- sharing private credentials
- committing secrets
- force-pushing without discussion
- changing architecture without confirmation
- adding unnecessary/generated files
- ignoring documentation updates
- mixing unsupported architecture patterns into the repo

---

## Architecture Discussion Rule

Before making changes to:

- backend framework
- AI service structure
- deployment platform
- database layer
- folder structure
- documentation strategy

Follow this flow:

```text
Inform team
Wait for confirmation
Then update the repo
```

---

## Professional Standard

The repo should stay clean, readable, and deployment-ready.

Every contributor should help maintain:

- clean folder structure
- updated documentation
- secure environment handling
- tested code
- clear pull requests

---

## Reporting Issues

Report project conduct issues to the project owner/team lead.
